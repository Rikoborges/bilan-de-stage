C'est quoi une API ?

API = Application Programming Interface. Une adresse web qui retourne des données.

Imagine un restaurant. Toi tu es le site web. Le serveur c'est l'API. La cuisine c'est le serveur distant avec les données. Tu commandes (tu envoies une requête HTTP), le serveur t'apporte le résultat (une réponse JSON). Tu n'as pas besoin de savoir comment c'est préparé en cuisine — tu reçois juste les données.

Le JSON (JavaScript Object Notation) c'est le format de la réponse. C'est du texte structuré avec des paires clé/valeur — facile à lire pour les machines et pour les humains.

exemple-api.js
// Tu envoies une requête à cette adresse (URL de l'API)
fetch('https://api.chucknorris.io/jokes/random')

// Tu reçois ce JSON en réponse :
{
  "id":    "abc123",
  "value": "Chuck Norris peut compiler sans erreur.",
  "url":   "https://api.chucknorris.io/jokes/abc123"
}

Créer un plugin WordPress

Un plugin permet d'ajouter des fonctionnalités à WordPress sans modifier son code source.

WordPress scanne automatiquement le dossier wp-content/plugins/ au démarrage. Pour chaque sous-dossier, il lit le premier fichier PHP et cherche un commentaire d'en-tête spécial. Si il le trouve, le plugin apparaît dans la liste d'administration. C'est tout — pas besoin d'enregistrement ou de configuration supplémentaire.

1
Créer le dossier : wp-content/plugins/mon-plugin-api/
2
Créer le fichier : mon-plugin-api.php
3
Ajouter l'en-tête PHP — WordPress le lit pour identifier le plugin
4
Aller dans Admin → Extensions → Activer
mon-plugin-api.php
<?php
/*
 * Plugin Name: Mon Plugin API
 * Description: Affiche une blague Chuck Norris via shortcode
 * Version:     1.0
 * Author:      Ricardo Borges — Riko Dev Studio
 */

// Sécurité : si quelqu'un accède directement au fichier PHP,
// on bloque. ABSPATH est une constante définie par WordPress.
if ( ! defined( 'ABSPATH' ) ) {
    exit;
}

Pourquoi ça marche ? WordPress lit le commentaire en haut du fichier PHP. Il reconnaît Plugin Name: et affiche le plugin dans l'admin. Sans ce commentaire, WordPress ignore le fichier.

Créer un shortcode

Un shortcode est une balise courte qu'on place dans une page WP pour exécuter du PHP.

Sans shortcode, le PHP s'exécute dans les fichiers du thème — difficile à gérer pour un client. Avec un shortcode, le client écrit [blague_api] dans l'éditeur WordPress et le plugin fait tout le travail. C'est propre, réutilisable et le client ne voit jamais de code.

Important : la fonction doit toujours return le HTML, jamais echo. Si on utilise echo, le contenu s'affiche en haut de la page au mauvais endroit.

shortcode-simple.php
// 1. La fonction qui retourne le HTML
function mon_shortcode() {
    return '<p>Bonjour depuis mon plugin !</p>';
    // ATTENTION : return, pas echo !
}

// 2. Enregistrer le shortcode dans WordPress
add_shortcode( 'bonjour', 'mon_shortcode' );
// [bonjour]  →  affiche : Bonjour depuis mon plugin !

// Shortcode avec paramètre : [saluer nom="Ricardo"]
function shortcode_saluer( $atts ) {
    $atts = shortcode_atts([
        'nom' => 'visiteur'
    ], $atts);
    return '<p>Bonjour, ' . $atts['nom'] . ' !</p>';
}
add_shortcode( 'saluer', 'shortcode_saluer' );

Appeler une API dans WordPress

On utilise wp_remote_get() — la fonction WordPress pour faire des requêtes HTTP en PHP.

On pourrait utiliser la fonction PHP native file_get_contents(), mais WordPress fournit wp_remote_get() qui est plus sécurisée, gère les timeouts, et s'intègre mieux avec le système de cache. Elle retourne un tableau avec le statut HTTP, les headers et le body (le contenu brut — le JSON).

Après avoir reçu le body, on le décode avec json_decode() pour accéder aux données propriété par propriété. Et esc_html() sécurise le texte avant de l'afficher (protection XSS).

mon-plugin-api.php — fonction complète
function afficher_blague() {

    // 1. Envoyer la requête HTTP GET
    $response = wp_remote_get('https://api.chucknorris.io/jokes/random');

    // 2. Vérifier si une erreur réseau s'est produite
    if ( is_wp_error( $response ) ) {
        return '<p>Erreur : API indisponible.</p>';
    }

    // 3. Extraire le texte brut de la réponse
    $body = wp_remote_retrieve_body( $response );

    // 4. Décoder le JSON → objet PHP
    $data = json_decode( $body );

    // 5. Retourner le HTML sécurisé
    return '<div class="blague"><p>'
         . esc_html( $data->value )
         . '</p></div>';
}

add_shortcode( 'blague_api', 'afficher_blague' );
// Utilisation dans une page WP : [blague_api]

Comprendre json_decode()

Transformer du texte JSON en objet PHP que le code peut lire.

L'API retourne une chaîne de texte brut — une string. On ne peut pas accéder directement aux données dedans. json_decode() transforme cette string en un objet PHP, ce qui nous permet d'accéder à chaque propriété avec la notation -> (comme les points en JavaScript).

Avec true en deuxième paramètre, on obtient un tableau associatif au lieu d'un objet — on accède alors aux données avec des crochets ['clé']. Les deux fonctionnent, c'est une question de préférence.

json-decode.php
// Ce qu'on reçoit de l'API — une simple string (texte brut)
$json = '{"value":"Chuck Norris...","id":"abc123"}';

// ── Mode Objet (défaut) ──────────────────────────────
$data = json_decode( $json );
echo $data->value;   // "Chuck Norris..."
echo $data->id;      // "abc123"

// ── Mode Tableau (avec true) ─────────────────────────
$data = json_decode( $json, true );
echo $data['value'];  // "Chuck Norris..." — même résultat

API dans un site HTML/JS normal

Même logique qu'en PHP — mais avec fetch() directement dans le navigateur.

fetch() est la fonction native JavaScript pour faire des requêtes HTTP. Elle retourne une Promise — une promesse que les données arriveront dans le futur. On utilise async/await pour écrire le code de manière lisible, ou .then() pour chaîner les étapes.

En React/Vite, on utilise la même logique dans un useEffect — qui s'exécute quand le composant se charge. On stocke les données dans un useState pour que React mette à jour l'affichage automatiquement.

script.js — site HTML statique
// Avec async/await — le plus lisible
async function chargerBlague() {
    const res  = await fetch('https://api.chucknorris.io/jokes/random');
    const data = await res.json();
    document.getElementById('blague').textContent = data.value;
}

// ── Dans React/Vite ──────────────────────────────────
const [blague, setBlague] = useState('');

useEffect(() => {
    fetch('https://api.chucknorris.io/jokes/random')
        .then( r => r.json() )
        .then( data => setBlague(data.value) );
}, []); // [] = s'exécute une seule fois au chargement

// Affichage JSX : <p>{blague}</p>

Le problème CORS

Pourquoi certaines APIs bloquent les appels depuis le navigateur.

CORS (Cross-Origin Resource Sharing) est une protection du navigateur. Par défaut, un site web ne peut pas appeler un serveur d'un autre domaine — c'est pour empêcher un site malveillant d'envoyer des requêtes en ton nom. L'API doit explicitement autoriser les appels externes avec un header spécial. Si elle ne le fait pas → erreur CORS.

Solution : passer par le backend. PHP avec wp_remote_get() n'est pas soumis à CORS — c'est une restriction du navigateur, pas d'HTTP en général. Node.js côté serveur non plus. On fait l'appel en PHP, et PHP retourne les données au navigateur.

Des APIs gratuites pour tes clients

Des fonctionnalités dynamiques à intégrer dans n'importe quel site — sans coût.

Une valeur ajoutée qu'on peut offrir à un client : rendre son site vivant — météo en temps réel, carte interactive, formulaire de contact sans backend complexe. Ces APIs sont gratuites pour un usage normal et s'intègrent en quelques lignes de code.

OpenWeatherMap

Météo en temps réel par ville. Idéal pour restaurants, hôtels, commerces locaux.

openweathermap.org/api

Google Maps

Carte interactive avec localisation exacte. Gratuit jusqu'à 28 000 req/mois.

maps.googleapis.com

EmailJS

Formulaire de contact sans backend. Gratuit jusqu'à 200 emails/mois.

api.emailjs.com

LRCLIB

Paroles de chansons synchronisées. Utilisé dans KaraokêParty.

lrclib.net/api

exchangerate-api

Taux de change EUR/BRL/USD en temps réel.

exchangerate-api.com
